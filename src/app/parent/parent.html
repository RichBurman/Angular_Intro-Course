@Input & @Output — How data flows

Golden rule: think from the child's perspective.

  • @Input — data coming IN to the child (parent → child)
  • @Output — events going OUT from the child (child → parent)

Step 1 — @Input

Square brackets pass data down into the child. The child declares @Input() count to receive it.

app.component.html
<app-counter [count]="countValue"></app-counter>

counter.component.ts
@Input() count: number = 0;

Step 2 — @Output

The child calls emit() to fire an event upward. It cannot change the parent's data directly.

counter.component.ts
@Output() countIncremented = new EventEmitter<number>();

increment() {
  this.count++;
  this.countIncremented.emit(this.count);
}

Step 3 — Parent reacts

Round brackets listen for the event. $event carries the emitted value. The parent updates its own data, which flows back down via @Input.

app.component.html
<app-counter (countIncremented)="onCountIncremented($event)"></app-counter>

app.component.ts
onCountIncremented(newCount: number) {
  this.countValue = newCount;
}

Summary: The parent owns the data. The child displays it via @Input and requests changes via @Output.
